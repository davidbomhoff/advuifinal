UI Design × AI — A Field Guide

The Designer's
Edge

Design should feel human, as it's built to service humans. A machine cannot understand how a design feels when used — but it can do the heavy lifting so you can focus on exactly that.

scroll to play your way through

It's easy to understand why the advent of AI might strike a chord with designers — it's a literal problem-solving machine. Despite this, a human hand in design will always be necessary. After studying design and exploring the potential of AI tools, I've come to understand there's a happy medium. Vibecoding an entire project produces something messy and ugly. Designing entirely by hand is beautiful but limited. The sweet spot is using AI for what it's good at.

This guide is hands-on. Each section is a mini-game that teaches you the concept by having you do it yourself. Let's say you're designing an online card game — you have the Figma prototype, but gameplay simulation can't be built in Figma. This is how you bridge that gap.

Styles → Variables → CSS

In Figma, your design decisions live as named styles. These are your design variables — the single source of truth for your visual language. The first step bridging Figma and code is translating these variables into CSS custom properties. When a style maps to a CSS variable, one change propagates everywhere — colors, spacing, font sizes, border radii, all of it.

The card game UI on the right is built entirely from design variables. Click any value in the CSS file to open its editor — change a color, drag a slider, and watch every element that references that variable update simultaneously. This is exactly how variable-driven design systems work in production.

Creating Logic with AI

This is where AI earns its place. You've designed a card game — beautiful layouts, polished animations — but game logic is the hard part: shuffling, dealing, scoring, state. Rather than figuring out algorithms yourself, you describe the behavior you want and let AI write the implementation.

The key is writing a good prompt. A great code prompt has four ingredients: Context (your stack and situation), Goal (what to build), Constraint (guardrails), and Format (how the output should look). Drag the pieces below into their correct slots — then build and score your prompt.

Figma → Code via MCP

Traditionally, turning a Figma design into code meant manually inspecting layer properties, copying values, and writing HTML by hand. Figma's MCP (Model Context Protocol) server changes this entirely — it lets an AI agent read your Figma file directly, inspect every layer, variable, and component, and generate production-ready code without you copying a single value.

Walk through the pipeline below step by step. Each step is clickable — see the exact commands, AI conversations, and outputs at each stage of turning your card game Figma file into a running app.

Walkthrough — Figma MCP Pipeline
🔁 Figma MCP Pipeline — click each step to walk through the process Step 0 / 5
01
Install the Figma MCP Server MCP
The Figma MCP server is a local bridge between your AI agent and the Figma API. Install it once and it runs as a background service your AI can talk to.
terminal
# Install the Figma MCP server globally npm install -g @figma/mcp-server # Add your Figma access token figma-mcp config --token YOUR_FIGMA_TOKEN # Start the server (runs on localhost:3845) figma-mcp start # ✓ Figma MCP server running on port 3845 # ✓ Connected to Figma API as: your@email.com
02
Connect AI Agent to MCP AI Agent
Point your AI agent (Claude, Cursor, Windsurf, etc.) at the MCP server. The agent can now call Figma tools: read files, inspect components, extract variables.
claude_desktop_config.json
{ "mcpServers": { "figma": { "command": "figma-mcp", "args": ["serve"], "env": { "FIGMA_TOKEN": "your-token-here" } } } } // Claude now has access to tools: // • figma_get_file(fileId) // • figma_get_node(fileId, nodeId) // • figma_get_styles(fileId) // • figma_get_components(fileId)
03
Extract Variables & Components Figma
Ask the AI to read your Figma file. It calls the MCP tools to pull design variables (colors, type, spacing) and component structure directly — no copy-pasting from the inspector.

Generate Code from Design AI Agent
With the component structure in context, ask the AI to generate HTML, CSS, and JS. It uses the exact values from your Figma file — no approximations.
