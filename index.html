Design should feel human, as it’s built to service humans. It’s easy to understand why the advent of AI might strike a chord with designers; it’s a literal problem-solving machine. Despite this, a human hand in design will always be necessary, as a machine cannot understand how a design feels when used by a human. After studying design and exploring the potential of AI tools, I’ve come to understand that there is a happy medium for designers when it comes to using AI. While it’s easy to just vibecode an entire project; process to prototype, the product will turn out messy and ugly. When designed by hand, you’ll get a great product; but maybe not reaching the full potential of what it could be. This tool looks to help UI designers understand how they can use the power of AI to turn a Figma prototype into an interactive app. Let’s say, for example, you’re designing an online card game. You have the vector files, stylesheets, and a pretty good Figma prototype, but simulation of gameplay is a function that cannot be built out in Figma, so the real user experience isn’t quite replicated. This is where I teach you how to upgrade your existing prototype to something that closer represents the MVP. All this will take is some basic knowledge of html, css, and js. First, let’s talk about converting Figma styles to css. This can be done by hand or using AI to create style “tokens”. These tokens are presets for css stylesheets and can be used throughout a file in liu of remembering specific colors, fonts, even spacings! The interactive activity below shows how a set of colors would look as css tokens. The next part is where AI is most useful for designers: coding complex mechanics. You know what you want in the prototype, but might not have the coding skills to quickly whip up a working script. The key is writing a good prompt. A great AI prompt for code has four ingredients: context (what you're building), goal (what you need), constraints (your stack), and format (how you want the output). The interactive activity below shows how a good prompt is built from knowledge on three code problems. Finally we got to the big one; Figma to code. Figma and html/css are more similar than one would think, they just speak different languages. The activity below shows how elements from a figma file can be rewritten into code, with or without AI assistance. These tools will add another level to your prototypes, improve dev handoff, and help to realize the designer’s vision in a more complex manner.
